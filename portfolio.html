Каталог авто з адмінкою всередині Telegram

Оголошення жили в переписці й нотатках: фото окремо, характеристики окремо, актуальність невідома. Клієнт питав «а що є до 20 тисяч?» — і продавець гортав чат руками.

Що побудовано

  • Картка авто з 13 полів: марка, модель, рік, двигун, КПП, привід, пробіг, ціна, телефон, VIN, локація, опис і кілька фото в галереї
  • Пошук для покупця — за маркою, роком, ціною та комбінований; неактивні оголошення не показуються
  • Адмін-панель у тому ж боті: додати авто покроково, відредагувати будь-яке поле, зняти з продажу одним натисканням, видалити разом із фото
  • Своя база SQLite з двома таблицями та зв'язком фото → авто (каскадне видалення, без «сиріт» у базі)
  • Міграція схеми на льоту — нові поля дописуються в наявну базу без втрати вже доданих авто
  • Окремий переглядач бази для власника: одна команда — і видно весь стан бази в консолі

Технічні рішення, які тут важливі

  • Оновлення полів іде через білий список дозволених колонок — назва поля ніколи не потрапляє в SQL напряму
  • Усі налаштування в .env: токен, ID адміна, шлях до бази, валюта. Файл створюється автоматично при першому запуску з підказками
  • Робота з базою через контекстний менеджер — з'єднання закривається навіть при помилці

Фрагмент коду

# оновлення поля лише з дозволеного списку — захист від ін'єкцій def update_field(car_id, field, value): allowed = {"brand", "model", "year", "price", "mileage", ...} if field not in allowed: raise ValueError(f"Заборонене поле: {field}")

Монітор X (Twitter) з розпізнаванням адрес контрактів

У ніші, де рішення ухвалюють за хвилини, людина фізично не може сидіти й оновлювати стрічку десятка акаунтів. Пост, помічений через півгодини, вже нічого не вартий.

Що побудовано

  • Сканування кожні 30 секунд — новий пост відстежуваного акаунта прилітає в Telegram майже миттєво
  • Автоматичний пошук адрес контрактів у тексті: Solana (base58 з перевіркою довжини ключа) та EVM-мережі (0x + 40 hex) — Ethereum, BSC, Polygon, Arbitrum
  • Фільтри по типу активності: твіти, цитати, ретвіти, відповіді, контракти — вмикаються окремо для кожного акаунта
  • Керування зі смартфона — додати чи прибрати акаунт, перемкнути фільтр можна інлайн-кнопками в Telegram, без доступу до сервера
  • Захист від дублів: ID кожного обробленого поста зберігається, повторних сповіщень не буває
  • Авторизована сесія через збережені cookies — видно й ті пости, що недоступні гостям

Складне місце проєкту

  • X віддає стрічку через внутрішні запити, а не готовим HTML. Скрейпер запускає справжній Chromium і перехоплює мережеві відповіді сторінки — тому не ламається від зміни верстки
  • Асинхронна архітектура: цикл сканування й Telegram-бот працюють паралельно в одному процесі, не блокуючи один одного
  • Валідація адрес перед відправкою — щоб у сповіщення не летіли випадкові рядки, схожі на base58

Запис клієнтів без жодного повідомлення майстру

Майстер узгоджував кожен запис у переписці під час роботи: «а на четвер є вікно?» — треба відкласти інструмент, подивитись зошит, відповісти. Накладки й забуті записи були питанням часу.

Що побудовано

  • Запис у чотири кроки: послуга → день → вільний час → ім'я і телефон. Клієнт нікого не чекає
  • Розрахунок вільних слотів за тривалістю послуги — манікюр 60 хв, педикюр 90, комбо 150. Система показує лише той час, куди послуга реально влазить до кінця робочого дня
  • Захист від накладок — зайнятий інтервал зникає зі списку одразу, двоє клієнтів не сядуть на одну годину
  • Панель майстра: відкрити робочі дні й години, редагувати список послуг та їх тривалість, бачити всі записи, скасувати будь-який
  • Сповіщення про кожен новий запис приходить майстру в Telegram миттєво
  • Клієнт бачить і скасовує свої записи сам — без дзвінків і незручних повідомлень
  • Київський часовий пояс з коректним фолбеком, якщо на сервері немає бази часових зон

Як я пишу код

Проєкт має пережити передачу замовнику

Ці правила однакові в усіх трьох проєктах вище — їх видно прямо в коді.

Ключі не живуть у коді

Токени, ID та шляхи — у файлі налаштувань поза репозиторієм. Проєкт можна показувати й передавати, не розкриваючи доступів.

Оновлення не ламає дані

Схема бази мігрує сама: нові поля дописуються в робочу базу, вже накопичена інформація лишається на місці.

Запуск з нуля за три команди

Список залежностей, автостворення конфіга з підказками, інструкція в шапці файлу. Проєкт підніме будь-який розробник.

Помилка не зупиняє систему

Таймаути й збої обробляються з повторною спробою, з'єднання з базою закривається навіть при виключенні.

Дані перевіряються

Введення користувача й знайдені в тексті адреси валідуються перед записом чи відправкою — сміття не потрапляє далі.

Керування без консолі

Власник керує системою з телефона через звичайні кнопки. Щоб змінити налаштування, не потрібен розробник.

Розбір задачі й оцінка безкоштовні. Якщо автоматизація не окупиться, скажу прямо.
